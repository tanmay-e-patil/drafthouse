## Architecture

Collaborative markdown editor. Rust backend (nanoservices pattern, single monolith binary), TanStack Start frontend, CRDT via Yrs.

### Nanoservice Pattern

Every feature is split across three crates:

| Layer | Responsibility |
|---|---|
| `networking` | HTTP/WS handlers, route registration (Actix-web) |
| `core` | Business logic, validation, orchestration |
| `dal` | SQL/CQL queries, DB interactions |

`ingress/src/main.rs` is the binary entry point — registers all routes from all networking crates.

### Services

| Service | DB | What it owns |
|---|---|---|
| `auth` | Postgres | Register, login, JWT, password reset, email verify, GDPR |
| `documents` | Postgres | Doc metadata, permissions, invite links, WS tickets |
| `collab` | ScyllaDB | WebSocket, Yrs CRDT sync, awareness, snapshots |

### Key Shared Crates

- `dal/kernel` — all domain structs (`User`, `Document`, `DocMember`, `InviteLink`, `WsTicket`, `MemberRole`). Read this to understand the entire domain.
- `dal/dal` — DAL trait definitions + `SqlxPostGresDescriptor` / `ScyllaDescriptor` impls
- `crates/utils` — `NanoServiceError`, `ResponseError` (RFC 7807), `safe_eject!`
- `crates/dal-tx-impl` — `#[impl_transaction]` proc macro

### Macros (Do Not Add New Ones)

| Macro | Use |
|---|---|
| `#[impl_transaction]` | Implement DAL trait for `SqlxPostGresDescriptor` or `ScyllaDescriptor` |
| `define_dal_transactions!` | Define DAL traits (Postgres + Scylla) |
| `safe_eject!` | Standardized error conversion |
| `#[api_endpoint]` | HTTP handler wrapper (JWT, RBAC, session cache) |
| `#[subscribe_to_event]` / `publish_event!` | In-process event bus |

### Cross-Service Events (In-Process, No Broker)

- `TitleUpdated { doc_id, title }` — documents → collab (broadcasts to WS room)
- `ExportRequested { user_id, email }` — auth → export handler (ZIP + email)

### CRDT / WebSocket

Active documents live in `DocStore: DashMap<DocId, DocRoom>`. One room per doc in process memory, shared `Arc<RwLock<yrs::Doc>>` across all WS connections. WebSocket uses y-websocket binary protocol.

**WS auth flow:** Client calls `POST /documents/:id/ws-ticket` → 30-second single-use token → `WS /collab/:doc_id?ticket=<token>`. Ticket stored hashed, burned on upgrade.

Snapshots trigger every 100 ops OR 30 seconds. Last 5 retained (ring buffer on `version INT 1-5`). WAL in ScyllaDB `ops` table has 7-day TTL — no cleanup needed.

### Frontend

Feature-Sliced Design (FSD):

```
frontend/
├── app/              # TanStack Start config, root layout
├── pages/            # Route components
├── widgets/editor/   # CodeMirror 6 + Yjs + toolbar + presence
├── features/
│   ├── collab/       # WS lifecycle, Yrs client, awareness
│   ├── auth/         # JWT refresh, token storage (Zustand memory only)
│   └── documents/    # CRUD, sharing modal, invite links
├── entities/
└── shared/
    └── api/generated/  # openapi-ts output (committed, do not hand-edit)
```

JWT: access token in Zustand (memory only, never localStorage). Refresh token as `httpOnly + SameSite=Strict` cookie. TanStack Query global `retry` handler calls `POST /auth/refresh` on 401.

### Testing Philosophy

- `auth`/`documents` core: unit tests with `MockDb` (generated by `#[impl_transaction]`). Zero DB.
- `collab` core: integration-heavy — CRDT bugs hide in mocks. Use real ScyllaDB via `testcontainers-rs`.
- Test external behavior (inputs, outputs, side effects). Do not assert on internal implementation details or private state.

### API

All errors are RFC 7807 via `ResponseError` on `NanoServiceError` — no per-handler boilerplate. After any endpoint change, run `make gen` to regenerate `shared/api/generated/` and commit the result.

### Migrations

Never run at app startup — CI/CD only. Postgres: `sqlx migrate` from `migrations/postgres/`. ScyllaDB: custom Rust runner from `migrations/scylla/*.cql`. All CQL uses `IF NOT EXISTS` (idempotent).
